Секция 1: Стек и структура проекта

Монорепо с двумя приложениями и общими типами

Структура репозитория

cumsino/ (GitHub repo)
📁 client/    ← Vite + React + TypeScript
  src/components/    ← UI компоненты (экраны игры)
  src/hooks/        ← useSocket, useGame
  src/store/        ← Zustand (состояние клиента)
📁 server/    ← Node.js + Express + Socket.IO
  src/game/        ← GameEngine, RoundManager
  src/socket/      ← обработчики событий
  questions.json   ← банк вопросов
📁 shared/    ← общие TypeScript типы
  types.ts         ← GameState, Player, Question…

Технологии

Client
Vite + React + TypeScript
TailwindCSS — стили
Framer Motion — анимации фишек
Socket.IO client — реалтайм
Zustand — глобальный стейт
Деплой: GitHub Pages
Server
Node.js + Express + TypeScript
Socket.IO — WebSocket сервер
In-memory state — игровое состояние
(никакой БД не нужно)

Деплой: Render.com (бесплатно)

Как подключаются игроки

Игрок открывает сайт на GitHub Pages
↓
Вводит имя + код комнаты
↓
WebSocket соединение с Render.com
↓
Все 4–7 игроков в одной игровой сессии
