Секция 3: Модель данных и события Socket.IO

Что хранится на сервере и как клиент с ним общается

Игровое состояние (in-memory, сервер)

Game
id: string
phase: GamePhase
roundIndex: number
lastMainMode: 'all' | 'gladiator'
currentQuestion: Question
gladiatorId?: string
players: Player[]
phaseTimer: number
Player
id: string (socket.id)
name: string
coins: number
currentBet: number
betTarget?: 'win' | 'lose'
  (только для гладиатора)
answer?: string | number | string[]
hasAnswered: boolean
Question (JSON-банк)
id: string
mode: 'all'|'gladiator'|'closest'|'top5'
topic: string
text: string
options?: string[]   // A/B/C/D
answer: string | number | string[]
numericAnswer?: number
items?: string[]     // для топ5
GamePhase (enum)
LOBBY
ANNOUNCE    ← тема/режим
BETTING     ← ставки
QUESTION    ← вопрос+ответ
REVEAL      ← итоги раунда
LEADERBOARD
GAME_OVER

События Socket.IO

Client → Server
join_game { name, gameCode }
place_bet { amount, target? }
submit_answer { answer }
start_game ← от первого игрока
Server → Client (broadcast)
game_state { phase, players, question… }
phase_changed { phase, timeLeft }
bet_updated { playerId, amount }
player_answered { playerId }
round_results { deltas[], winners[] }
game_over { winner }
Принцип: один большой game_state
При каждой смене фазы сервер шлёт полный game_state — клиент просто рендерит то, что пришло.
Мелкие события (bet_updated, player_answered) — только для живых анимаций (фишки, галочки).
Так клиент никогда не теряет синхронизацию.
