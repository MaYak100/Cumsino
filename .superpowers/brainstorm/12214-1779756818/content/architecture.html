Архитектурные подходы

Выбери подход для реализации мультиплеерной викторины

A

Монорепо: Vite + React / Express + Socket.IO ⭐ Рекомендую

Два отдельных приложения в одном репозитории.

Структура
/client → Vite + React (деплой на GitHub Pages)
/server → Node.js + Express + Socket.IO
         (деплой на Render.com бесплатно)
/shared → общие типы и константы

Плюсы

  • Простой стек, много туториалов
  • Socket.IO — идеален для игр
  • Frontend и backend независимы
  • Бесплатный деплой

Минусы

  • Render.com засыпает (15 мин простоя)
  • Нужно управлять двумя приложениями
B

Full-stack: Next.js + WS через Ably/Pusher

Next.js обслуживает и фронт, и API. WebSockets через внешний сервис.

Структура
/app → Next.js (всё в одном)
  /api → REST + WS через Ably SDK
  /components → React компоненты
Деплой: Vercel (бесплатно)

Плюсы

  • Один деплой (Vercel)
  • Не засыпает

Минусы

  • Ably — внешняя зависимость
  • Игровое состояние — где хранить?
  • Serverless плохо держит сессию
C

Всё локально + ngrok-тоннель

Сервер на твоей машине, ngrok пробрасывает публичный URL для друзей.

Структура
/client → Vite + React (билд раздаёт сервер)
/server → Node.js + Socket.IO
ngrok → публичный URL для друзей
Деплой: никакого, запускаешь сам

Плюсы

  • Нет ограничений по сну
  • Полный контроль
  • Самый простой деплой

Минусы

  • Твой ПК должен быть включён
  • ngrok бесплатно — случайный URL
  • Нет постоянной ссылки
