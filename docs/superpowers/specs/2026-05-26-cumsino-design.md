# Cumsino — Викторина со ставками для друзей

**Дата:** 2026-05-26  
**Стек:** Vite + React + TypeScript / Node.js + Express + Socket.IO  
**Деплой:** GitHub Pages (client) + Render.com бесплатный тир (server)

---

## 1. Обзор

Браузерная мультиплеерная викторина с экономикой перераспределения в стиле казино. 4–7 игроков подключаются с разных компьютеров через интернет. Каждый получает стартовый капитал 500 фишек. Игра заканчивается, когда кто-то достигает 3000 фишек. Новые фишки появляются только в спец-режимах; основные режимы перераспределяют существующий капитал.

## 4. Игровые режимы

### 4.1 ВОПРОС ДЛЯ ВСЕХ (основной)

1. **Анонс:** показывается только тема вопроса (~5 сек)
2. **Ставки вслепую:** игроки ставят фишками из своего банка в общий котёл (~30 сек)
3. **Текст вопроса:** появляется сам вопрос без вариантов ответа (~5 сек)
4. **Варианты ответа:** открываются 4 варианта A/B/C/D, начинается таймер (~40 сек)
5. **Итог:** проигравшие теряют ставку → пул делится между победителями пропорционально их ставкам

**Фишка:** если все ответили верно — никто ничего не заработал. Если верно ответил один — забирает весь пул.

### 4.2 ГЛАДИАТОР (основной)

1. **Выбор:** рандомный игрок становится Гладиатором
2. **Толпа видит вопрос и правильный ответ** → делает ставки в две корзины: «ОН ОТВЕТИТ» / «ОН ЗАВАЛИТ»; Гладиатор видит заглушку (~30 сек)
3. **Текст вопроса:** появляется у Гладиатора без вариантов (~5 сек)
4. **Варианты ответа:** открываются у Гладиатора; толпа в реальном времени видит, какой вариант он сейчас наводит курсором (~40 сек)
5. **Итог для толпы:** угадавшая группа делит пул проигравшей группы пропорционально ставкам
6. **Итог для Гладиатора:** +300 фишек из системы за верный ответ, 0 за неверный

### 4.3 КТО БЛИЖЕ (спец-режим, +фишки в пул)

- Вопрос с числовым ответом
- Нет фазы ставок
- Кто ближе к правильному ответу → **+200 фишек** из системы
- Точное попадание → дополнительно **+200 фишек**
- При ничьей пул делится между всеми победителями поровну

### 4.4 ТОП 5 (спец-режим, +фишки в пул)

- Расставить 5 элементов в правильном порядке
- Нет фазы ставок
- **+20 фишек** за каждую верную позицию
- Все 5 верно → **+150 фишек** (вместо 100)

## 5. Логика последовательности раундов

```
Раунд 1  → ВОПРОС ДЛЯ ВСЕХ (гарантировано)
Раунд 2  → ГЛАДИАТОР (гарантировано)
Раунд 3+ → 70% противоположный основной режим от последнего
           30% спец-режим (выбирается случайно из доступных)
```

«Противоположный» = если последним основным был ВОПРОС → следующий ГЛАДИАТОР, и наоборот.

## 6. Алгоритм разбивки выигрыша на фишки

Балансы хранятся как целые числа, кратные 10. При отображении выигрышей и распределении пула сервер вычисляет `ChipBreakdown` — набор фишек для анимации.

**Принцип:** крупная часть суммы (≥70%) покрывается большими номиналами (500, 100), оставшаяся часть (≈30%) — мелкими (50, 20, 10). Это создаёт визуально смешанную стопку фишек.

```typescript
// shared/chips.ts
const DENOMS = [500, 100, 50, 20, 10] as const
type ChipBreakdown = Record<typeof DENOMS[number], number>

function decomposeToChips(amount: number): ChipBreakdown {
  const chips: ChipBreakdown = { 500: 0, 100: 0, 50: 0, 20: 0, 10: 0 }
  let remaining = Math.round(amount / 10) * 10

  if (remaining >= 200) {
    // Резервируем ~20% для мелких фишек (не менее 20)
    const smallBudget = Math.max(20, Math.round(remaining * 0.2 / 10) * 10)
    const largePart = remaining - smallBudget

    // Крупная часть — 500 и 100
    chips[500] = Math.floor(largePart / 500)
    remaining -= chips[500] * 500
    chips[100] = Math.floor(remaining / 100)
    remaining -= chips[100] * 100
  }

  // Остаток — мелкие номиналы
  chips[50] = Math.floor(remaining / 50)
  remaining -= chips[50] * 50
  chips[20] = Math.floor(remaining / 20)
  remaining -= chips[20] * 20
  chips[10] = Math.round(remaining / 10)

  return chips
}
```

`round_results` от сервера включает `chipBreakdown` для каждого игрока — клиент анимирует именно эти фишки, летящие к/от стопки игрока.

## 7. Модель данных

```typescript
// shared/types.ts

type GamePhase =
  | 'LOBBY'
  | 'ANNOUNCE'
  | 'BETTING'
  | 'QUESTION_TEXT'   // вопрос без вариантов, 5 сек
  | 'QUESTION'        // вопрос + варианты ответа
  | 'REVEAL'
  | 'LEADERBOARD'
  | 'GAME_OVER'

type GameMode = 'all' | 'gladiator' | 'closest' | 'top5'

interface Player {
  id: string           // socket.id
  name: string
  chips: number        // текущий баланс
  currentBet: number
  betTarget?: 'win' | 'lose'  // только для ГЛАДИАТОРА (толпа)
  answer?: string | number | string[]
  hasAnswered: boolean
}

interface Question {
  id: string
  mode: GameMode
  topic: string
  text: string
  options?: string[]           // A/B/C/D для основных режимов
  answer: string | number | string[]
  numericAnswer?: number       // для КТО БЛИЖЕ
  items?: string[]             // для ТОП 5 (перемешанный порядок игроку)
  orderedItems?: string[]      // для ТОП 5 (правильный порядок, только сервер)
}

interface GameState {
  id: string
  phase: GamePhase
  roundIndex: number
  lastMainMode: 'all' | 'gladiator'
  currentQuestion: Question | null
  gladiatorId?: string
  players: Player[]
  phaseTimeLeft: number
}
```

## 8. Socket.IO события

### Client → Server
| Событие | Данные | Описание |
|---------|--------|---------|
| `join_game` | `{ name, gameCode }` | вход в лобби |
| `start_game` | — | запуск (любой игрок) |
| `place_bet` | `{ amount, target? }` | ставка фишками |
| `submit_answer` | `{ answer }` | ответ на вопрос |
| `gladiator_hover` | `{ optionIndex \| null }` | гладиатор навёл курсор (throttle 200ms) |

### Server → Client
| Событие | Данные | Описание |
|---------|--------|---------|
| `game_state` | `GameState` | полный стейт при смене фазы |
| `phase_changed` | `{ phase, timeLeft }` | уведомление о новой фазе |
| `bet_updated` | `{ playerId, amount }` | для анимации фишек в корзинах |
| `player_answered` | `{ playerId }` | игрок ответил (без раскрытия ответа) |
| `gladiator_hovering` | `{ optionIndex \| null }` | броадкаст ховера гладиатора (только толпе) |
| `round_results` | `{ deltas: { playerId, delta, chipBreakdown }[] }` | изменения + раскладка фишек |
| `game_over` | `{ winner: Player }` | победитель |

**Принцип:** при каждой смене фазы — полный `game_state`. Мелкие события (`bet_updated`, `gladiator_hovering`) только для живых анимаций.

## 9. UI и визуальный стиль

**Тема:** казино — тёмно-зелёное сукно (`#0d1f0d`), золотые акценты (`#ffd700`), красный для негатива, яркий зелёный для позитива.

### Фишки (5 номиналов)
| Номинал | Цвет |
|---------|------|
| 10 | ⚪ Белая |
| 20 | 🟢 Зелёная |
| 50 | 🔵 Синяя |
| 100 | 🔴 Красная |
| 500 | ⚫ Чёрная |

**Механика ставок:** клик по фишке → анимация полёта к зоне ставки (Framer Motion), повторный клик → убрать последнюю добавленную. Кнопка «Подтвердить ставку» фиксирует выбор.

**Выигрыши:** фишки с анимацией летят к стопке игрока согласно `chipBreakdown` из `round_results`.

### Ключевые экраны
1. **Лобби** — название, код комнаты, список игроков, кнопка старта
2. **Анонс** — режим раунда, тема
3. **Ставки** — фишки, текущая ставка, таймер
4. **Гладиатор (толпа)** — имя гладиатора, вопрос+ответ, две корзины, ховер-индикатор
5. **Гладиатор (сам гладиатор)** — только вопрос с вариантами, без корзин толпы
6. **Текст вопроса** — вопрос без вариантов, 5-секундная пауза (QUESTION_TEXT)
7. **Вопрос + варианты** — текст, 4 варианта A/B/C/D (QUESTION)
8. **КТО БЛИЖЕ** — числовой инпут
9. **ТОП 5** — drag-and-drop сортировка 5 элементов
10. **Итоги раунда** — дельты фишек с анимацией стопок
11. **Таблица лидеров** — текущий рейтинг
12. **Конец игры** — победитель

## 11. Граничные случаи

- **Игрок ставит 0 фишек** (пропускает ставку): участвует в вопросе, но ничего не выигрывает из пула
- **У игрока меньше 10 фишек**: может поставить всё что есть (ва-банк), жёсткого минимума нет
- **Все ответили верно в ВОПРОС ДЛЯ ВСЕХ**: никто ничего не теряет, ставки возвращаются
- **Обе корзины в ГЛАДИАТОРЕ пусты**: раунд проводится, выплат нет — только бонус гладиатору

## 12. Расширяемость

- Спец-режимы регистрируются как отдельные модули в `server/src/game/modes/` — добавление нового режима не затрагивает основную логику
- `questions.json` может разбиться на несколько файлов по режимам
- Лимит победы (3000 фишек) — константа в `shared/constants.ts`
